# Agent Bootstrap

## Purpose
Provide minimal shared context for any new agent session in this repository without forcing task-specific scaffolding.

## Start Here (in order)
1. `AGENTS.md`
2. `FOLDER_MAP.md`
3. `docs/MARKDOWN_INDEX.md`
4. `docs/research/R_AND_D_LEDGER.md`
5. `experiments-local/registry/INDEX.md`
6. `docs/research/DECISION_GATES.md`
7. `docs/SESSION_CLOSEOUT_CHECKLIST.md`

## Working Rules
- Use registry traces for experiment context, not chat history.
- Use ledger entries for cross-session interpretation context.
- Add new run traces and ledger entries when meaningful work is done.

## What this does not do
- It does not prescribe task strategy or conclusions.
- It does not override the user’s current request.
